    # aHooks是如何解决React闭包问题的？

    # React的闭包问题

    先看一个例子：

    import React, { useState, useEffect } from "react";

export default () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    setInterval(() => {
      console.log("setInterval:", count);
    }, 1000);
  }, []);

  return (
    <div>
      count: {count}
      <br />
      <button onClick={() => setCount((val) => val + 1)}>增加 1</button>
    </div>
  );
};

    当点击按钮时，发现setInterval中打印出来的值并没有发生变化，始终都是0。这就是React的闭包问题。

    # 产生的原因

    为了维护Function Component的state，React用链表的方式存储Function Component中的hooks，并为每个hooks创建一个对象。

    type Hook = {
  memoizedState: any,
  baseState: any,
  baseUpdate: Update<any, any> | null,
  queue: UpdateQueue<any, any> | null,
  next: Hook | null,
};

    这个对象的memoizedState属性就是用来存储上一次组件更新后的state，next指向下一个hook对象。在组件更新的过程中，hook函数的 执行顺序是不变的，就可以根据这个链表拿到当前hooks对应的Hook对象，函数式组件就是这样有了state的能力。

    同时制定了一系列的规则，比如不能讲hooks写入到if...else...中，确保能够正确拿到相应hook的state。

    回到刚才那个例子：

    const [count, setCount] = useState(0);

useEffect(() => {
  setInterval(() => {
    console.log("setInterval:", count);
  }, 1000);
}, []);

    它第一次执行的时候，执行useState，count为0，执行useEffect，执行其中的回调逻辑，启动定时器，每隔1s输出setInterval:0。

    当我点击按钮让count增加1的时候，整个函数式组件重新渲染，这时前一个执行的链表已经存在了。useState将Hook对象上保存我的状态置为1， 那么此时count也为1了。执行useEffect，其依赖项为空，不执行回调函数。但是之前的回调函数还是在的，它还是会每隔1s执行console.log("setInterval:", count)， 但这里的count还是第一次执行时候的count值，因为在定时器的回调函数中被引用了，形成的闭包一直被保存。

    # 解决方法

    # 方法一

    给useEffect设置依赖项，重新执行函数，设置新的定时器，拿到最新值。

    useEffect(() => {
  if (timer.current) {
    clearInterval(timer.current);
  }
  timer.current = setInterval(() => {
    console.log("setInterval:", count);
  }, 1000);
}, [count]);

    # 方法二

    使用useRef。useRef返回一个可变的ref对象，其,current属性被初始化为传入的参数。

    useRef创建的是一个普通的JavaScript对象，而且会在每次渲染时返回同一个ref对象，当我们变化它的current属性时，对象的引用都是同一个，所以定时器能读到最新的值。

    const lastCount = useRef(count);

// 解决方法二
useEffect(() => {
  setInterval(() => {
    console.log("setInterval:", lastCount.current);
  }, 1000);
}, []);

return (
  <div>
    count: {count}
    <br />
    <button
      onClick={() => {
        setCount((val) => val + 1);
        // +1
        lastCount.current += 1;
      }}
    >
      增加 1
    </button>
  </div>
);

    # useRef=>useLatest

    回到aHooks主题，基于上面的第二种方案，useLatest这个hook随之诞生，它返回当前最新值的hook，可以避免闭包问题：

    import { useRef } from 'react';
// 通过 useRef，保持每次获取到的都是最新的值
function useLatest<T>(value: T) {
  const ref = useRef(value);
  ref.current = value;

  return ref;
}

export default useLatest;

    # useEvent=>useMemoizedFn

    React中的另外一个场景，是基于useCallBack的。

    const [count, setCount] = useState(0);

const callbackFn = useCallback(() => {
  console.log(`Current count is ${count}`);
}, []);

    上面的例子中，我们的count不管怎样变化，执行callbackFn打印出来的count值始终为0，这是因为回调函数被useCallback缓存形成闭包，

    那么我们怎样解决这个问题呢？官方提出了useEvent，它解决的问题：如何同时保持函数引用不变与访问到最新状态。使用它之后，上面变成了：

    const callbackFn = useEvent(() => {
  console.log(`Current count is ${count}`);
});

    实际上，在ahooks中已经实现了相似的功能，那就是useMemoizedFn。

    useMemoizedFn时吃句话function的hook，理论上，可以使用它完全的代替useCallback。使用useMemoizedFn，可以省略第二个参数deps， 同时保证函数的地址永远不会变化。

    我们看下它的源码：

    function useMemoizedFn<T extends noop>(fn: T) {
  // 通过 useRef 保持其引用地址不变，并且值能够保持值最新
  const fnRef = useRef<T>(fn);
  fnRef.current = useMemo(() => fn, [fn]);
  // 通过 useRef 保持其引用地址不变，并且值能够保持值最新
  const memoizedFn = useRef<PickFunction<T>>();
  if (!memoizedFn.current) {
    // 返回的持久化函数，调用该函数的时候，调用原始的函数
    memoizedFn.current = function (this, ...args) {
      return fnRef.current.apply(this, args);
    };
  }

  return memoizedFn.current as T;
}
