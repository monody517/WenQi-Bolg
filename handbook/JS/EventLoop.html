    前言

    # 前言

    JS是一门单线程语言，这就意味着所有的任务都需要排队，前一个任务结束才会执行下一个任务。这样会出现一个问题： 如果JS执行时间过长，就可能会造成页面渲染不连贯，导致页面阻塞甚至卡死。为了解决这个问题，JS中出现了同步和 异步。他们的本质区别是：一条流水线上的各个流程执行顺序不同。在执行顺序之前，先了解一下什么是同步任务与异步 任务。

    同步任务：主线程上的任务，按照顺序从上到下依次执行，前一个任务完成才会执行下一个任务。

    异步任务：不进入主线程，而是进入任务队列的任务，当主线程上的任务执行完成后，才会执行任务队列中的任务。

    # 宏任务，微任务

    宏任务，微任务属于js代码的范畴，js代码分为同步代码，异步代码，异步代码又分为宏任务和微任务。

    任务 宏/微任务 环境
    网络请求(Ajax) 宏任务 浏览器
    setTimeout()定时器 宏任务 浏览器
    fs.readFile()读取文件 宏任务 Node
    Promise.then() 微任务 浏览器/Node
    async/await 微任务 浏览器/Node

    # 事件循环EventLoop执行机制

    1. 进入到script标签，就进入到第一次事件循环
    2. 遇到同步代码，立即执行
    3. 遇到宏任务，放入到宏任务队列中
    4. 遇到微任务，放到微任务队列中
    5. 执行完成所有同步代码
    6. 执行微任务
    7. 微任务执行完成，本次队列清空
    8. 寻找下一个宏任务，重复步骤1

    依次反复直到清空所有宏任务，这种不断重复的执行机制，就叫做事件循环

    # 易错点

    1. promise本身是同步的代码，只有在它后面调用的then()方法才是微任务
    new Promise(resolve => {
   console.log(7)
   resolve
}).then(funciton () {
   console.log(8)
})
console.log(9)
// 7 9 8
    1. await中的代码才是微任务，右边的代码还是会立即执行
    console.log(1)
async function async1(){
    await async2()
   console.log(2)
}
async function async2(){
   console.log(3)
}
async1()
// 1 3 2
    1. script标签本身是宏任务，当页面中出现多个script标签时，浏览器会将script标签作为宏任务解析
    <scrpit>
   console.log(1)
   setTimeout(()=>{
   console.log(2)
   })
</scrpit>
<scrpit>
   console.log(3)
</scrpit>
<!--1 3 2-->
    1. setTimeout的误区

    宏任务setTimeout的回调不一定在指定的时间后能执行，而是在指定的时间后将回调函数放入事件循环的队列中。 如果时间到了JS引擎还在执行同步任务，这个回调函数需要等待；如果当前事件循环的队列中还有其他回调，需要等 其他回调执行完。另外setTimeout 0ms也不是立刻执行，他有一个默认的最小时间4ms。
