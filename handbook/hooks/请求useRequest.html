    # useRequest简介

    useRequest是一个强大的异步数据管理Hooks，React项目中的网络请求场景使用useRequest就够了。官方文档 (opens new window)

    useRequest通过插件式组织代码，目前已有的能力包括：

    1. 自动请求/手动请求
    2. 轮询
    3. 防抖
    4. 节流
    5. 屏幕聚焦重新请求
    6. 错误重试
    7. loading delay
    8. SWR(stale-while-revalidate)
    9. 缓存

    # 架构

    如上图，我将useRequest分为了几个模块

    1. 入口 useRequest。它负责的是初始化处理数据以及将结果返回。
    2. Fetch。是整个 useRequest 的核心代码，它处理了整个请求的生命周期。
    3. plugin。在 Fetch 中，会通过插件化机制在不同的时机触发不同的插件方法，拓展 useRequest 的功能特性。
    4. utils 和 types.ts。提供工具方法以及类型定义。

    # useRequest 入口处理

    先从入口文件开始，packages/hooks/src/useRequest/src/useRequest.ts

    function useRequest<TData, TParams extends any[]>(
  service: Service<TData, TParams>,
  options?: Options<TData, TParams>,
  plugins?: Plugin<TData, TParams>[],
) {
  return useRequestImplement<TData, TParams>(service, options, [
    ...(plugins || []),
    useDebouncePlugin,
    useLoadingDelayPlugin,
    usePollingPlugin,
    useRefreshOnWindowFocusPlugin,
    useThrottlePlugin,
    useAutoRunPlugin,
    useCachePlugin,
    useRetryPlugin,
  ] as Plugin<TData, TParams>[]);
}

export default useRequest;

    这里有三个参数，service(请求参数)，options(配置参数)以及官方文档中没有提到的plugins(插件列表)。

    可以看到整体返回了useRequestImplement方法，主要是对fetch类进行实例化。

    const update = useUpdate();
// 保证请求实例都不会发生改变

  // 目前只有 useAutoRunPlugin 这个 plugin 有这个方法

    serviceRef, // 请求实例

    重点关注的是几个属性，一个是state，类型为FetchState<TData,Tparams>，一个是pluginlmpls，他是PluginReturn<TData,TParams>数组，实际上这个属性用来存所有网点插件执行后返回的结果。还有一个count属性，为number类型。

    先看FetchState<TData,Tparams>的定义，他定义在src/types里：

    export interface FetchState<TData, TParams extends any[]> {
  loading: boolean;
  params?: TParams;
  data?: TData;
  error?: Error;
}

    看起来是保存一个请求结果的上下文信息，这些信息需要暴露给外部用户使用，如loading，errors，data都是我们在请求中需要使用的信息。

    const { data, error, loading } = useRequest(service);

    而对应的Fetch封装了setState API，实际上就是用来更新state数据：

    setState(s: Partial<FetchState<TData, TParams>> = {}) {
    this.state = {
      ...this.state,
      ...s,
    };
    this.subscribe();
  }

    这里除了设置state的值外，还调用了一个subscribe方法，它的作用是通过subscribe调用通知useRequestImplement组件重新渲染。

    再看下PluginReturn<TData,TParams>的类型定义：

    export interface PluginReturn<TData, TParams extends any[]> {
  onBefore?: (params: TParams) =>
    | ({
        stopNow?: boolean;
        returnNow?: boolean;
      } & Partial<FetchState<TData, TParams>>)
    | void;

  onRequest?: (
    service: Service<TData, TParams>,
    params: TParams,
  ) => {
    servicePromise?: Promise<TData>;
  };

  onSuccess?: (data: TData, params: TParams) => void;
  onError?: (e: Error, params: TParams) => void;
  onFinally?: (params: TParams, data?: TData, e?: Error) => void;
  onCancel?: () => void;
  onMutate?: (data: TData) => void;
}

    实际上都是一些回调钩子，从名字上看对应了请求的各个阶段，除了onMutate是其内部扩展的钩子。也就是说pluginlmpls里存的是一系列含有钩子函数的对象合集，所以这里其实就是一个回调的合集，里面是不同的请求阶段的回调。

    要搞清楚Fetch的运作模式，只需要看两个核心API的实现：runPluginHandler和runAsync，其他的API都是调用这两个API再做一些额外的特殊逻辑处理。

    # runPluginHandler

    先看runPluginHandler：

    runPluginHandler(event: keyof PluginReturn<TData, TParams>, ...rest: any[]) {
  // @ts-ignore
  const r = this.pluginImpls.map((i) => i[event]?.(...rest)).filter(Boolean);
  return Object.assign({}, ...r);
  }

    这个方法的实现就是接收一个event参数，类型为keyof PluginReturn<TData, TParams>，也就是onBefore | onRequest | onSuccess | onError | onFinally | onCancel | onMutate的联合类型，以及其他一些额外参数，然后从pluginImpls中找出所有对应的event回调钩子函数并执行，拿到结果并返回。

    同时你会发现基本所有的插件功能都是在一个请求或多个阶段中实现的，也就是说我们只需要在请求的相应阶段执行插件的逻辑就能完成插件的功能。

    # runAsync

    再看runAsync的实现：

      async runAsync(...params: TParams): Promise<TData> {
    this.count += 1;
    const currentCount = this.count;

    const {
      stopNow = false,
      returnNow = false,
      ...state
    } = this.runPluginHandler('onBefore', params);

    // stop request
    if (stopNow) {
      return new Promise(() => {});
    }

    this.setState({
      loading: true,
      params,
      ...state,
    });

    // return now
    if (returnNow) {
      return Promise.resolve(state.data);
    }

    this.options.onBefore?.(params);

    try {
      // replace service
      let { servicePromise } = this.runPluginHandler('onRequest', this.serviceRef.current, params);

      if (!servicePromise) {
        servicePromise = this.serviceRef.current(...params);
      }

      const res = await servicePromise;

      if (currentCount !== this.count) {

        return new Promise(() => {});
      }

      this.setState({
        data: res,
        error: undefined,
        loading: false,
      });

      this.options.onSuccess?.(res, params);
      this.runPluginHandler('onSuccess', res, params);

      this.options.onFinally?.(params, res, undefined);

      if (currentCount === this.count) {
        this.runPluginHandler('onFinally', params, res, undefined);
      }

      return res;
    } catch (error) {
      if (currentCount !== this.count) {
        // prevent run.then when request is canceled
        return new Promise(() => {});
      }

      this.setState({
        error,
        loading: false,
      });

      this.options.onError?.(error, params);
      this.runPluginHandler('onError', error, params);

      this.options.onFinally?.(params, undefined, error);

      if (currentCount === this.count) {
        this.runPluginHandler('onFinally', params, undefined, error);
      }

      throw error;
    }
  }

    看下来其实只进行了一些简单操作。调用传入的获取数据的方法，拿到成功或失败的结果，进行数据处理后更新到state，执行插件的各回调钩子或通过options传入的回调函数。
