    什么是koa

    # 什么是koa

    koa和express一样，都是node.js的一个web开发框架。他通过利用async函数，帮助我们摒弃掉了回调函数的写法，并且有力的增强了错误处理。为了保持简洁，koa没有绑定任何的中间件，而是通过优雅的方式去引入，帮助我们愉快的编写服务端代码。

    # 简单使用

    Koa应用是一个包含一组中间件函数的对象，它是按照类似对战的方式组织和执行的。Koa一个关键的设计点在于其低级中间件层中提供高级‘语法糖’，包括一些诸如内容协商，缓存清理，代理支持和重定向等常见任务的方法。但是Koa并没有捆绑这些中间件，所以Koa仍保持了一个很小的体积。 我们来看最简单的Hello world应用：

    const Koa = require('koa');
const app = new Koa()

app.use(async ctx => {
    ctx.body = 'Hello World'
})

app.listen(3000)

    # 级联

    Koa中间件以更传统的方式级联，也就是我们说的洋葱模型，请求从最外层进去，又从最里层出来，我们直接看一个例子：

    const Koa = require('koa')
const app = new Koa()

app.use(async (ctx, next)=>{
  let startTime = new Date().getTime()
  await next()
  let endTime = new Date().getTime()
  console.log(`此次的响应时间为：${endTime - startTime}ms`)
})

app.use(async (ctx, next) => {
  console.log('111, 然后doSomething')
  await next()
  console.log('111 end')
})

app.use(async (ctx, next) => {
  console.log('222, 然后doSomething')
  await next()
  console.log('222 end')
})

app.use(async (ctx, next) => {
  console.log('333, 然后doSomething')
  await next()
  console.log('333 end')
})

app.listen(3333, ()=>{
  console.log('server is running at http://localhost:3333')
})

    我们可以发现，333是不执行的，因此中间件的顺序对next的执行有很大的影响。

    # 路由

    当项目功能多起来后，路由处理相应的也多了起来，app.js的代码就会这样：

    const Koa = require('koa');

const app = new Koa();

app.use(async ctx => {
  const { path } = ctx
  if (path === '/a') {
    // 功能 A
    ctx.body = 'a';
  } else if (path === '/b') {
    // 功能 B
    ctx.body = 'b';
  } else if (path === '/c') {
    // 功能 C
    ctx.body = 'c';
  } else {
    ctx.body = 'hello world'
  }
});

app.listen(3333, ()=>{
    console.log('server is running at http://localhost:3333')
})

    这种将所有路由和路由处理放在一起的写法，随着代码量的增多，后期会难以维护，因此我们将路由处理部分从启动文件app.js中拿出来，维护一个router文件， 并用第三方路由管理插件koa-router来管理路由，我们在router文件夹下写路由处理部分的代码:

    const koaRouter = require('koa-router');
const router = new koaRouter();

router.get('/a', ctx => {
  ctx.body = 'a'
});

router.get('/b', ctx => {
  ctx.body = 'b'
});

router.get('/c', ctx => {
  ctx.body = 'c'
});

module.exports = router;

    再修改app.js的代码，路由处理从router中引入：

    const Koa = require('koa');

const router = require('./router')

const app = new Koa();

const port = '8082'
const host = '0.0.0.0'

app.use(router.routes());
/*
原先当路由存在，请求方式不匹配的时候，会报 404，
加了这个中间件，会报请求方式不被允许
*/
app.use(router.allowedMethods());


app.listen(port, host, () => {
console.log(`API server listening on ${host}:${port}`);
});

    到目前为止，我们创建了一个路由文件，但是路由文件中，我们希望的情况是路由文件中只处理路由相关，至于逻辑相关应该放在逻辑相关文件中处理， 因此我们再创建一个router-handler文件夹专门用来处理具体业务逻辑。这个文件里我们需要一个统一的入口，因此我们创建index.js，再创建一个test.js文件。

    app/router-handler/test.js专门处理具体业务逻辑：

    const list = async ctx => {
    ctx.body = '路由改造'
}

module.exports = {
    list
}

    app/router-handler/index,js专门处理业务逻辑导出：

    const test = require('./test');

module.exports = {
    test
}

    app/router/routes.js专门创建路由：

    const {test} = require('../router-handler');

const routes = [
    {
        method: 'get',
        path: '/a',
        controller: test.list
    }
]

module.exports = routes

    app/router/index.js也要做出相应的修改：

    const koaRouter = require('koa-router');
const router = new koaRouter();

const routeList = require('./routes');

routeList.forEach(item => {
  const { method, path, controller } = item;
  //  router 第一个参数是 path， 后面跟上路由级中间件 handler（上面编写的路由处理函数）
  router[method](path, controller);
});

module.exports = router;
